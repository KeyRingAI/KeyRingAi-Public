# Reporting Vulnerabilities

Please report suspected vulnerabilities privately. Do not open public GitHub issues for security bugs, credential exposure, privacy concerns, abuse paths, suspected leaks, or exploit details.

Security contact: support@keyringlabs.com

Recommended subject line: `SECURITY: <short summary>`

## What To Include

Include a clear summary, affected product area, impact, safe reproduction steps, relevant screenshots or logs with secrets removed, app version if applicable, operating system, provider involved if relevant, and your preferred contact information.

Do not send real provider API keys, license keys, passwords, customer data, private prompts, proprietary files, private logs, or credentials unless the KeyRing AI team explicitly requests them through a private channel.

## Responsible Disclosure Expectations

Give the team time to investigate before public disclosure. Avoid testing against accounts, systems, data, or users you do not own or have permission to test. Do not attempt persistence, data destruction, credential theft, social engineering, denial of service, or access to customer data.

If you discover private material in this public repository, report it privately and do not redistribute it.

## Public Repository Scope

This repository contains documentation and public developer resources only. It does not contain the proprietary application source code. Security reports about the public repository should focus on exposed secrets, accidental proprietary material, unsafe documentation, or public-facing process issues.

## Public Boundary

This document provides a reporting process. It does not create a bug bounty program, safe-harbor contract, or permission to test systems outside your authorization.
