## Example Review Checklist

Before adding any example, confirm that it contains:

- No proprietary application source code
- No copied private repo files or history
- No secrets, tokens, certificates, credentials, or `.env` files
- No customer data
- No internal infrastructure details
- Only public, reviewed API or integration behavior
- A clear license notice if the example code is intended to be MIT licensed
