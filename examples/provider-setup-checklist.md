# Provider Setup Checklist

This checklist helps users configure a provider cleanly and avoid common bring-your-own-key mistakes.

## Provider Account

Create or select the provider account you want to use with KeyRing AI. Confirm that API access is enabled, billing or credits are active if required, and the models you plan to use are available to the account.

## API Key

Create a dedicated API key for KeyRing AI where the provider supports it. Prefer scoped keys, spending limits, and usage alerts. Store the key only in the app's provider settings. Do not paste it into prompts, presets, attachments, screenshots, or support messages.

## KeyRing AI Setup

Open API Settings. Select the provider. Paste the key. Save changes. Confirm that the provider status changes from missing to configured or active.

## Validation

Send a short prompt to a known model with default settings. If it fails, check the visible error, API Settings, provider dashboard, and provider service status.

## Model Inventory

Open Provider Manager. Review the saved model list. Run Model Discovery if available. Save the provider configuration after reviewing the discovered models and mode mappings.

## Public Boundary

This checklist describes public workflow steps only. It does not include private provider credentials, source code, or internal validation logic.
